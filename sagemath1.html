Installing SageMath on Ubuntu 18.04 part 1

Lately I have decided that I wanted to move away from Wolfram Mathematica tool set, although part of my present research rely on it. I was feeling that there was not many positive aspects to it when compared to competitors like Maple and SageMath, other than the fact that I can get it quite cheaply at the moment. That will certainly change in the future, so I decided to plan and move away from this tooling and migrate to a more open envorinment. That was why I decided to go with SageMath.

Motivations aside, what really triggered me was the fact I wanted to set up a org file in Emacs with most of my work including numerics and symbolic calculations and export it to pdf or html^1. Orgmode really hits the nail at this one but it was really saddening to see the state of ob-mathematica^2, I had a really bad time with it although I did get to work with Jupyter Notebooks. One of the most positive aspects of SageMath was the fact that both sage-shell-mode and ob-sage exist, I was pretty excited to start working on getting started on it.

Now for the installation. This version of Ubuntu was really odd to install SageMath at, I went looking for places to start and readily found their website and binaries for linux. After downloading I installed in the usual way, by first unpacking the tarball 

tar -xvjf sage-9.0-Ubuntu_18.04-x86_64.tar.bz2

Then executing with 

./sage

But unfortunetely I had problems permission problems installing this way. I thought then that I had been lucky to find this tutorial, following it though and making sure to replace the deprecated command 

gksu gedit /usr/local/bin/sage

with 

gedit admin:///usr/local/bin/sage

Resulted in same as above. Then finally I head to the good & old package manager 

sudo apt install sagemath

Still no joy. Will I be able to install SageMath to replace Mathematica in anytime soon? Let us hope so.
